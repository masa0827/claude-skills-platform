## 1. エグゼクティブサマリー

### 事業概要

AI Skills Libraryは、日本のビジネスパーソン向けにAIプロンプトテンプレート（「スキル」）を提供するWebプラットフォームである。ChatGPT・Claude・Gemini・Copilotの4ツールに対応し、営業・マーケティング・CS・バックオフィス・企画リサーチの5カテゴリ、約50スキルを収録している。

### 現状

| 項目 | 状態 |
|------|------|
| スキル数 | 46スキル / 5カテゴリ |
| 収益 | 0円（全スキル無料公開） |
| 月間固定コスト | 約125円（Vercel Hobby + Supabase Free + ドメイン費） |
| 技術基盤 | Next.js 16 / React 19 / Tailwind CSS 4 / TypeScript / Supabase / Claude API |
| SEO基盤 | sitemap / robots.txt / OGP / JSON-LD / Google Search Console / GTM |
| 法務基盤 | 利用規約（有料・マーケットプレイス対応済み）/ 特商法 / プライバシーポリシー |
| AIレコメンド | Claude API連携のチャットベース推薦機能 |
| ユーザー認証 | 未実装 |
| 決済機能 | 未実装 |

### 収益化方針

**プラットフォーム手数料モデル**を軸とする。

1. まず自社制作のプレミアムスキルで有料販売を開始（単品販売）
2. 販売・決済の仕組みが回り始めた段階で外部クリエイターに出品を開放
3. クリエイター売上の20%をプラットフォーム手数料として徴収

既存の46スキルは当面無料で維持し、トラフィック獲得エンジンとして活用する。

### 12ヶ月後の目標

| 指標 | 目標値 |
|------|--------|
| MAU | 15,000 |
| 有料スキル数 | 30個（自社15 + 外部15） |

| MRR（プラットフォーム収益） | 40,000円 + 広告収益30,000円 = 70,000円 |

## 2. 市場分析

### 2.1 日本のAIツール市場

日本の生成AI市場は2025年に約3,000億円規模（IDC Japan推計）に達し、2028年には1兆円を超える見通しである。ただし、実際の業務活用は「使い方がわからない」という壁で停滞している企業が多い。

**AI Skills Libraryが狙うのは、この「使い方がわからない」層に対する実用テンプレートの提供である。**

### 2.2 プロンプトマーケットプレイス市場

| サービス | 地域 | 特徴 | 月間トラフィック（推定） |
|----------|------|------|--------------------------|
| PromptBase | 海外 | 最大手、画像生成中心 | 300万 |
| AIPRM | 海外 | Chrome拡張、ChatGPT特化 | 200万 |
| PromptHero | 海外 | 画像生成プロンプト中心 | 150万 |
| プロンプト工房 | 日本 | 日本語特化、個人運営 | 数千 |

**日本市場の状況:**
- 海外のプロンプトマーケットは画像生成AI中心で、ビジネス業務向けは手薄
- 日本語に最適化されたビジネス特化のプロンプトマーケットは事実上存在しない
- ChatGPTの日本月間ユーザーは推定1,500万人超、しかし業務活用率は低い

### 2.3 ポジショニング

```
                高品質（プロ向け）
                    |
                    |  ★ AI Skills Library
                    |  （ビジネス業務特化 x 日本語ネイティブ）
                    |
    無料 ─────────────────────── 有料
                    |
                    |
                    |  海外マーケット
                    |  （画像生成中心 x 英語）
                    |
               低品質（汎用）
```

**差別化要因:**
1. **日本のビジネス文脈に完全最適化**: 敬語、稟議文化、日本の商慣行を前提とした設計
2. **Before/After付き実例**: 使用前後が具体的にイメージできる
3. **マルチAI対応**: 特定ツールにロックインしない
4. **品質管理済み**: 1スキルあたりcontent + examples + promptTemplate + steps + tipsの5層構造

## 6. リスクと対策

### 6.1 主要リスク

| # | リスク | 影響度 | 発生確率 | 対策 |
|---|--------|--------|---------|------|
| 1 | **有料スキルが売れない** | 高 | 中 | まず300円の低価格で市場検証。売れなければ無料スキルの「プレミアム版」（Before/Afterの充実版）として再設計 |
| 2 | **SEOで流入が伸びない** | 高 | 中 | 記事10本で3ヶ月検証し効果なければX/note/Qiitaのコンテンツマーケに注力 |
| 3 | **クリエイターが集まらない** | 中 | 高 | 初期は自社制作で回し、売れた実績をもってクリエイターを口説く。最悪、自社のみでも成立する設計にする |
| 4 | **AIツール側が類似機能を提供** | 高 | 低 | 「日本のビジネス文脈特化」という差別化を維持。AI側の公式プロンプトは汎用的で、業界特化はカバーしにくい |
| 5 | **Stripe/Supabaseの料金体系変更** | 中 | 低 | 代替サービス（Square/Firebase Auth）への移行計画を事前に調査 |
| 6 | **無料スキルのコピー・転売** | 低 | 高 | テキスト自体の保護は困難。「品質の継続的アップデート」と「信頼性のあるブランド」で差別化 |

### 6.2 競合参入リスク

| 競合タイプ | 脅威度 | 当社の防御策 |
|-----------|--------|------------|
| 海外プロンプトマーケット（日本語対応） | 中 | 日本のビジネス文化を前提とした設計は容易に模倣できない |

### 7.3 損益分岐の考え方

この事業の最大の利点は**固定コストが極めて低い**こと。月間コスト7,000円を超えれば黒字であり、有料スキル15件/月（平均500円）の販売で達成できる。

リスクは金銭的損失ではなく**機会費用**（自分の時間をこの事業に投じることの価値）である。そのため、フェーズ2終了時点（6ヶ月目）で有料スキルの需要が確認できなければ、早期にピボットまたは撤退の判断を行う。

### 8.2 外注の判断基準

基本方針は「すべて自社（1-2名）で実行」だが、以下の場合のみ外注を検討:

- SEO記事の量産が必要で自社工数が足りない場合 → ライター外注（1本3,000-5,000円）
- 法務面の確認が必要な場合 → スポットで弁護士相談（1回30,000円程度）

## 9. 成功指標と18ヶ月後の判断

| 判断基準（12ヶ月目） | アクション |
|----------------------|-----------|
| MRR 100,000円以上 | スケール継続。広告投資の開始を検討 |

**キャッシュカウとしての最終形態:**
- 月間作業5-10時間（クリエイター審査 + メルマガ配信 + 軽微なメンテナンス）
- MRR 50,000〜100,000円が自動的に入る状態
- 外部クリエイターが自発的にスキルを出品し、コンテンツが自動増殖する構造

## 付録: 技術実装の優先順位

### 収益化に直結する実装（Must）

1. **Supabase Auth**: Google OAuth + メール認証。購入履歴の紐付けに必須
2. **Stripe Checkout**: 単品購入のCheckout Session + Webhook。サブスクは不要（単品販売モデル）
3. **購入済みスキル管理**: Supabaseのpurchasesテーブル + RLS。購入済みスキルのコンテンツアクセス制御
4. **メールキャプチャ**: Supabaseのnewsletterテーブル + フッターフォーム

### 収益を伸ばす実装（Should）

6. **スキルセット販売**: 複数スキルのバンドル商品。客単価向上

8. **クリエイター管理画面**: 出品 + 売上確認 + 振込依頼

### あると良い実装（Nice to have）
