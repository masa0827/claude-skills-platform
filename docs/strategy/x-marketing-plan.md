- 日本はX利用者数 世界第2位（約6,700万人）
- ビジネス層のピーク: 7:00-8:30（通勤）、12:00-13:00（昼休み）、20:00-22:00（帰宅後）
- 金曜夕方は「週末にやってみよう」の行動喚起に最適
- 月曜朝は「今週の目標」系コンテンツの反応が高い

### 1.4 トーン＆マナー

#### 基本方針

| 要素 | 方針 |
|------|------|
| 一人称 | 使わない（サービス主語 or 主語なし） |
| 語尾 | 「〜です」「〜ます」のですます調。ただしカジュアル要素も混ぜる |
| 専門用語 | AI初心者にもわかる言葉。英語用語には必ず日本語補足 |
| 絵文字 | 控えめに使用。1投稿あたり2〜3個まで。ビジネス感を損なわない |
| ハッシュタグ | 投稿末尾に2〜3個。本文中には入れない |
| 数字 | 積極的に使用。「45分→5分」のような具体的時短数値 |

#### NGワード・表現

| NG | 理由 | 代替表現 |
|----|------|----------|
| AIプロンプト | ニッチすぎる、一般層に響かない | AIスキル、AI活用テンプレ |
| プロンプトエンジニアリング | 技術者向け用語 | AI活用術、AIの使い方 |
| 爆速 / 神ツール | 煽り系は信頼性を損なう | 実際の時短数値で示す |
| 🔥🚀 | ビジネス層には不適切な印象 | ✅💡📊 など落ち着いた絵文字 |

#### トーンの具体例

```
◎ 良い例:
「営業提案書、毎回ゼロから書いてませんか？
PREP法ベースのテンプレを使えば、45分かかっていた作業が5分に。
しかも論理構成が崩れない。」

✕ 悪い例:
「🔥営業提案書が爆速で完成する神プロンプト🔥
マジで使わないと損！！」
```

## 3. バイラル戦略

### 3.1 バイラルコンテンツパターン

#### パターン1: Before/After型（最重要）

**構造:**
```
【Before / After】〇〇スキル

Before:
→ 従来のやり方（苦痛ポイント）

After:
→ AI活用後（時短数値）

▼ 試してみる
[URL]
```

**効果的な理由:**
- 時短数値が具体的 → 説得力
- 共感（Before）→ 驚き（After）の感情変化
- リツイートしやすい（「これ便利」という文脈で共有される）

**量産リスト:**

| スキル | Before | After | 時短数値 |
|--------|--------|-------|----------|
| 営業提案書 | 構成で悩んで2時間 | PREP法で論理的な提案書 | 45分→5分 |
| FAQ自動生成 | 問い合わせを1件ずつ整理 | JSON-LD付きFAQ一括生成 | 4時間→15分 |
| 経費仕訳 | 勘定科目で毎回検索 | 科目＋仕訳＋消費税区分を即判定 | 30分→3分 |
| 市場規模推定 | 断片データの手動計算 | TAM/SAM/SOM＋感度分析 | 半日→15分 |
| メールシーケンス | 2通目以降が書けない | 5通の目的＋文面＋タイミング設計 | 60分→10分 |
| SOP作成 | ベテランへのヒアリング | 手順＋判断基準＋例外処理を構造化 | 1日→30分 |
| 事業計画書 | 何を書けばいいか不明 | 金融公庫形式の全章ドラフト | 1週間→2時間 |

#### パターン4: スレッド型（深掘り）

**最適な使い方:**
- カテゴリー別スキル紹介（営業5選、バックオフィス8選 など）
- 1スキルの使い方ステップバイステップ解説
- AI活用のコツ・まとめ系

**スレッド設計テンプレート:**
```
1/N [フック: 共感 or 問題提起]
↓
2/N〜(N-1)/N [各スキル or 各ステップの紹介]
↓
N/N [まとめ＋CTA（サイトURL）]
```

**効果的な理由:**
- スレッドはXアルゴリズムで優遇される（滞在時間が長い）
- 保存（ブックマーク）されやすい → 後日の再訪問
- 1ツイートでは伝えきれない情報量を発信可能

#### コミュニティ参加のルール

1. **売り込まない**: いきなりリンクを貼らない。まず価値提供（質問への回答、Tips共有）
2. **3:1の法則**: コミュニティへの貢献3回に対して、自社サービスの言及は1回まで
3. **文脈に合わせる**: 「〇〇で困ってます」→「うちのスキルで解決できますよ」は自然な流れ
4. **継続する**: 1回限りの投稿ではなく、週1〜2回の定期参加
